## Run 005 — Region Map Generation

**Date:** 2026-06-30  
**Script:** `code/04_build_region_maps.py`

### Goal

Generate semantic region maps for every ROHE raw sample so that later LLaVA and epistemic masking experiments can separate visual tokens into:

1. removed-object region,
2. context region,
3. background region.

These region maps will later be used for region-wise causal masking experiments.

### Input

Raw ROHE dataset:

```text
data/rohe_raw/
```

Each sample was expected to contain:

```text
original.jpg
removed.png
mask.png
metadata.json
```

### Method

For each sample:

1. Loaded the COCO object mask from `mask.png`.
2. Resized the mask to `336 × 336` using nearest-neighbor interpolation.
3. Used the resized COCO mask as the exact removed-object region.
4. Created the context region by dilating the removed-object mask and subtracting the original removed-object mask.
5. Created the background region as all pixels not belonging to removed or context regions.
6. Mapped the image into a `24 × 24` CLIP patch-token grid.
7. Assigned each patch token to one of:
   - `removed`
   - `context`
   - `background`
8. Saved token-to-region mappings and region masks.

### Important Parameters

```text
IMAGE_SIZE = 336
PATCH_SIZE = 14
GRID_SIZE = 24
EXPECTED_TOKENS = 576
DILATION_SIZE = 35
OVERLAP_THRESHOLD = 0.25
```

### Output

```text
outputs/region_maps_rohe/
```

For each sample, the script generated:

```text
removed_mask.png
context_mask.png
background_mask.png
token_to_region.json
region_counts.json
metadata.json
```

It also generated:

```text
outputs/region_maps_rohe/manifest_region_maps.json
outputs/region_maps_rohe/skipped_region_maps.json
```

### Result

```text
Samples found: 636
Successful samples: 636
Skipped samples: 0
```

Every successful sample had:

```text
total = 576
```

This confirms that every image was correctly mapped to the expected `24 × 24 = 576` CLIP patch-token grid.

### Raw Log

```text
logs/run_04_build_region_maps.txt
```

### Notes

The removed-object region is based on the exact COCO instance mask.

The context region is not manually annotated; it is defined automatically as a dilation ring around the removed object.

The background region is everything outside the removed and context regions.

This run does not yet create the final common dataset. It creates region maps for all raw ROHE samples. The final common dataset will be created after quality filtering.

### Method

The script performs the following steps:

1. Reads `outputs/rohe_quality.csv`.
2. Selects only rows where `good_sample == True`.
3. Clears any existing final dataset folders:
   - `data/rohe_final/`
   - `outputs/region_maps_final/`
4. Copies each selected raw sample from:
   - `data/rohe_raw/sample_xxxxxx/`
   to:
   - `data/rohe_final/sample_xxxxxx/`
5. Copies the corresponding region maps from:
   - `outputs/region_maps_rohe/sample_xxxxxx/`

6. Creates a final manifest listing all selected samples.

**Date:** 2026-06-30  
**Script:** `code/07_prepare_hpc_inputs.py`

### Goal

Prepare the final common ROHE dataset for HPC-based LLaVA and epistemic uncertainty experiments.

This step converts the final dataset into a simple structure that can be uploaded to the cluster and used by evaluation scripts.

### Input

Final common dataset:

```text
data/rohe_final/
```

Final region maps:

```text
outputs/region_maps_final/
```

Each final sample was expected to contain:

```text
original.jpg
removed.png
mask.png
mask_overlay.png
metadata.json
```

Each final region-map folder was expected to contain:

```text
token_to_region.json
region_counts.json
metadata.json
removed_mask.png
context_mask.png
background_mask.png
```

### Method

For each final sample:

1. Copy the original image into a flat `original_images/` folder.
2. Copy the removed image into a flat `removed_images/` folder.
3. Convert the removed `.png` image into `.jpg` format for compatibility with the Epistemic repository attack script.
4. Copy token-region files into a flat `token_regions/` folder.
5. Read the sample metadata.
6. Extract the target object and question.
7. Create JSONL question files for original-image and removed-image evaluation.
8. Create a manifest for all prepared HPC inputs.

### Output

HPC input folder:

```text
outputs/hpc_inputs/
```

Generated folders:

```text

outputs/hpc_inputs/removed_images/

outputs/hpc_inputs/token_regions/
```

Generated question files:

```text

outputs/hpc_inputs/questions_removed.jsonl
outputs/hpc_inputs/questions_removed_jpg.jsonl
```

Generated manifest:

```text
outputs/hpc_inputs/hpc_manifest.json
```

### Expected Labels

For original images:

```text

```

because the target object is still present.

For removed images:

```text
label = no
```

because the target object has been removed.

### Result

```text
HPC inputs prepared.
Prepared samples: 522
Skipped samples: 0
```

### Raw Log

```text
logs/run_07_prepare_hpc_inputs.txt
```

### Notes

The `questions_original.jsonl` file will be used to check whether LLaVA can correctly recognize the target object when it is present.

The `questions_removed.jsonl` file will be used to check whether LLaVA hallucinates the target object after removal.

The `removed_images_jpg/` folder is included because the original Epistemic attack pipeline expects `.jpg` images in some modes.

This run does not execute LLaVA or generate uncertainty masks. It only prepares clean HPC-ready inputs.
